# Alert Component

Displays a callout for important messages that attract user attention.

---

## Overview

The Alert component provides a way to display important messages that attract user attention. It follows accessibility guidelines with proper ARIA attributes and supports two visual variants.

---

## Package

- **Package:** BlazorUI.Components
- **Namespace:** BlazorUI.Components.Alert
- **Source:** https://github.com/blazorui-net/ui/tree/main/src/BlazorUI.Components/Components/Alert

---

## Installation

```bash
dotnet add package BlazorUI.Components
```

Add to `_Imports.razor`:

```razor
@using BlazorUI.Components.Alert
```

---

## API Reference

### Main Component: `<Alert>`

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Variant` | `AlertVariant` | `Default` | Visual style variant (Default, Destructive) |
| `Class` | `string?` | `null` | Additional CSS classes |
| `Icon` | `RenderFragment?` | `null` | Optional icon to display |
| `ChildContent` | `RenderFragment?` | - | Alert content (AlertTitle, AlertDescription) |

### Sub-components

#### `<AlertTitle>`

Renders the alert title with appropriate styling.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Title text |

#### `<AlertDescription>`

Renders the alert description text.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Description text |

### Enum: `AlertVariant`

| Value | Description |
|-------|-------------|
| `Default` | Standard alert with neutral styling |
| `Destructive` | Red/error styling for critical messages |

---

## Basic Usage

```razor
<Alert>
    <AlertTitle>Heads up!</AlertTitle>
    <AlertDescription>
        You can add components to your app using the CLI.
    </AlertDescription>
</Alert>
```

---

## Examples

### Example 1: Default Alert

```razor
<Alert>
    <AlertTitle>Note</AlertTitle>
    <AlertDescription>
        This is an informational message.
    </AlertDescription>
</Alert>
```

### Example 2: Destructive Alert

```razor
<Alert Variant="AlertVariant.Destructive">
    <AlertTitle>Error</AlertTitle>
    <AlertDescription>
        Your session has expired. Please log in again.
    </AlertDescription>
</Alert>
```

### Example 3: Alert with Icon

```razor
@using BlazorUI.Icons.Lucide.Components

<Alert>
    <Icon>
        <LucideIcon Name="info" Class="h-4 w-4" />
    </Icon>
    <ChildContent>
        <AlertTitle>Information</AlertTitle>
        <AlertDescription>
            This feature is currently in beta.
        </AlertDescription>
    </ChildContent>
</Alert>
```

### Example 4: Destructive Alert with Icon

```razor
<Alert Variant="AlertVariant.Destructive">
    <Icon>
        <LucideIcon Name="circle-alert" Class="h-4 w-4" />
    </Icon>
    <ChildContent>
        <AlertTitle>Warning</AlertTitle>
        <AlertDescription>
            Your account is approaching its storage limit.
        </AlertDescription>
    </ChildContent>
</Alert>
```

## Styling

### Variants

```razor
<Alert Variant="AlertVariant.Default">Default styling</Alert>
<Alert Variant="AlertVariant.Destructive">Destructive/error styling</Alert>
```

### Custom Styling

```razor
<Alert Class="my-custom-class">
    <AlertTitle>Custom Styled</AlertTitle>
    <AlertDescription>With additional classes.</AlertDescription>
</Alert>
```

## Accessibility

- Uses `role="alert"` for screen reader announcements
- Icons inherit the alert's text color for visual consistency
- Semantic HTML structure with proper heading hierarchy
