# Alert Dialog Component

A modal dialog that interrupts the user with important content and expects a response.

---

## Overview

The AlertDialog component is used for critical confirmations that require user acknowledgement. Unlike regular dialogs, alert dialogs cannot be dismissed by clicking outside or pressing Escape - users must interact with the action buttons.

---

- **Package:** BlazorUI.Components


## API Reference

### Main Component: `<AlertDialog>`

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Open` | `bool?` | `null` | Controls open state in controlled mode |
| `OpenChanged` | `EventCallback<bool>` | - | Callback when open state changes |
| `DefaultOpen` | `bool` | `false` | Initial open state for uncontrolled mode |
| `ChildContent` | `RenderFragment?` | - | Child content |

### Sub-components

#### `<AlertDialogTrigger>`

Button that opens the dialog.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `ChildContent` | `RenderFragment?` | - | Trigger content (typically a Button) |

#### `<AlertDialogContent>`

The modal content container.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Dialog content |

#### `<AlertDialogHeader>`

Container for title and description.

#### `<AlertDialogTitle>`

The dialog title.

#### `<AlertDialogDescription>`

The dialog description text.

#### `<AlertDialogFooter>`

Container for action buttons.

#### `<AlertDialogAction>`

Confirms the action and closes the dialog.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `ChildContent` | `RenderFragment?` | - | Action button content |

#### `<AlertDialogCancel>`

Cancels and closes the dialog.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `ChildContent` | `RenderFragment?` | - | Cancel button content |

## Basic Usage

```razor
<AlertDialog>
    <AlertDialogTrigger>
        <Button Variant="ButtonVariant.Outline">Show Dialog</Button>
    </AlertDialogTrigger>
    <AlertDialogContent>
        <AlertDialogHeader>
            <AlertDialogTitle>Are you absolutely sure?</AlertDialogTitle>
            <AlertDialogDescription>
                This action cannot be undone. This will permanently delete your
                account and remove your data from our servers.
            </AlertDialogDescription>
        </AlertDialogHeader>
        <AlertDialogFooter>
            <AlertDialogCancel>
                <Button Variant="ButtonVariant.Outline">Cancel</Button>
            </AlertDialogCancel>
            <AlertDialogAction>
                <Button>Continue</Button>
            </AlertDialogAction>
        </AlertDialogFooter>
    </AlertDialogContent>
</AlertDialog>
```

## Examples

### Example 1: Destructive Action

```razor
<AlertDialog>
    <AlertDialogTrigger>
        <Button Variant="ButtonVariant.Destructive">Delete Account</Button>
    </AlertDialogTrigger>
    <AlertDialogContent>
        <AlertDialogHeader>
            <AlertDialogTitle>Delete Account</AlertDialogTitle>
            <AlertDialogDescription>
                Are you sure you want to delete your account? All of your data
                will be permanently removed. This action cannot be undone.
            </AlertDialogDescription>
        </AlertDialogHeader>
        <AlertDialogFooter>
            <AlertDialogCancel>
                <Button Variant="ButtonVariant.Outline">Cancel</Button>
            </AlertDialogCancel>
            <AlertDialogAction>
                <Button Variant="ButtonVariant.Destructive">Yes, delete account</Button>
            </AlertDialogAction>
        </AlertDialogFooter>
    </AlertDialogContent>
</AlertDialog>
```

### Example 2: Controlled State

```razor
<Button OnClick="@(() => _isOpen = true)">Open Dialog</Button>
<span>Dialog is: @(_isOpen ? "Open" : "Closed")</span>

<AlertDialog @bind-Open="_isOpen">
    <AlertDialogContent>
        <AlertDialogHeader>
            <AlertDialogTitle>Controlled Dialog</AlertDialogTitle>
            <AlertDialogDescription>
                This dialog's open state is controlled by a parent component.
            </AlertDialogDescription>
        </AlertDialogHeader>
        <AlertDialogFooter>
            <AlertDialogCancel>
                <Button Variant="ButtonVariant.Outline">Close</Button>
            </AlertDialogCancel>
        </AlertDialogFooter>
    </AlertDialogContent>
</AlertDialog>

@code {
    private bool _isOpen;
}
```

## Alert Dialog vs Dialog

| Feature | Alert Dialog | Dialog |
|---------|--------------|--------|
| Dismiss by clicking outside | No | Yes |
| Dismiss by pressing Escape | No | Yes |
| Must acknowledge with button | Yes | No |
| ARIA role | `alertdialog` | `dialog` |
| Use case | Destructive/critical confirmations | Forms, settings, general content |

## State Management

### Uncontrolled

```razor
<AlertDialog DefaultOpen="false">
    <!-- Component manages its own state -->
</AlertDialog>
```

### Controlled

```razor
<AlertDialog @bind-Open="isOpen">
    <!-- Parent controls the state -->
</AlertDialog>

@code {
    private bool isOpen;
}
```

## Accessibility

- Uses `role="alertdialog"` for proper screen reader announcements
- Focus is trapped within the dialog
- Cannot be dismissed accidentally (no click-outside or Escape)
- Proper ARIA attributes for title and description
