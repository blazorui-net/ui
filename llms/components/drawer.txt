# Drawer Component

Mobile-friendly panel that slides in from the edge of the screen.

---

## Overview

The Drawer component provides a sliding panel that can appear from any edge of the screen. It's optimized for mobile interfaces and includes touch-friendly interactions with an optional drag handle.

---

## Package

- **Package:** BlazorUI.Components
- **Namespace:** BlazorUI.Components.Drawer
- **Source:** https://github.com/blazorui-net/ui/tree/main/src/BlazorUI.Components/Components/Drawer

---

## Installation

```bash
dotnet add package BlazorUI.Components
```

Add to `_Imports.razor`:

```razor
@using BlazorUI.Components.Drawer
```

## API Reference

### Main Component: `<Drawer>`

Root container managing drawer state.

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Direction` | `DrawerDirection` | `Bottom` | Slide direction (Top, Right, Bottom, Left) |
| `Open` | `bool?` | `null` | Controlled open state |
| `OpenChanged` | `EventCallback<bool>` | - | Open state change callback |
| `DefaultOpen` | `bool` | `false` | Initial open state |
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Trigger and content |

### Sub-components

#### `<DrawerTrigger>`

Element that opens the drawer when clicked.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Trigger element |

The drawer panel container.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|

| `ChildContent` | `RenderFragment?` | - | Drawer content |

#### `<DrawerHeader>`

Header section of the drawer.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Header content |

#### `<DrawerFooter>`

Footer section of the drawer.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Footer content |

#### `<DrawerTitle>`

Drawer title text.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Title text |

#### `<DrawerDescription>`

Drawer description text.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Description text |

#### `<DrawerClose>`

Element that closes the drawer when clicked.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Close element |

### Enum: `DrawerDirection`

| Value | Description |
|-------|-------------|
| `Top` | Slides from top of screen |
| `Right` | Slides from right side |
| `Bottom` | Slides from bottom (default) |
| `Left` | Slides from left side |

## Basic Usage

```razor
<Drawer>
    <DrawerTrigger>
        <Button Variant="ButtonVariant.Outline">Open Drawer</Button>
    </DrawerTrigger>
    <DrawerContent>
        <DrawerHeader>
            <DrawerTitle>Edit Profile</DrawerTitle>
            <DrawerDescription>
                Make changes to your profile here.
            </DrawerDescription>
        </DrawerHeader>
        <div class="p-4">
            <p>Drawer content goes here...</p>
        </div>
        <DrawerFooter>
            <Button>Save changes</Button>
            <DrawerClose>
                <Button Variant="ButtonVariant.Outline">Cancel</Button>
            </DrawerClose>
        </DrawerFooter>
    </DrawerContent>
</Drawer>
```

## Examples

### Example 1: Different Directions

```razor
<Drawer Direction="DrawerDirection.Top">
    <DrawerTrigger>
        <Button>Top</Button>
    </DrawerTrigger>
    <DrawerContent ShowHandle="false">
        <DrawerHeader>
            <DrawerTitle>Top Drawer</DrawerTitle>
        </DrawerHeader>
    </DrawerContent>
</Drawer>

<Drawer Direction="DrawerDirection.Right">
    <DrawerTrigger>
        <Button>Right</Button>
    </DrawerTrigger>
    <DrawerContent ShowHandle="false">
        <DrawerHeader>
            <DrawerTitle>Right Drawer</DrawerTitle>
        </DrawerHeader>
    </DrawerContent>
</Drawer>

<Drawer Direction="DrawerDirection.Left">
    <DrawerTrigger>
        <Button>Left</Button>
    </DrawerTrigger>
    <DrawerContent ShowHandle="false">
        <DrawerHeader>
            <DrawerTitle>Left Drawer</DrawerTitle>
        </DrawerHeader>
    </DrawerContent>
</Drawer>
```

### Example 2: Controlled State

```razor
<Button OnClick="@(() => isOpen = true)">Open Drawer</Button>

<Drawer @bind-Open="isOpen">
    <DrawerContent>
        <DrawerHeader>
            <DrawerTitle>Controlled Drawer</DrawerTitle>
        </DrawerHeader>
        <DrawerFooter>
            <Button OnClick="@(() => isOpen = false)">Close</Button>
        </DrawerFooter>
    </DrawerContent>
</Drawer>

@code {
    private bool isOpen = false;
}
```

## Behavior

- Displays overlay backdrop when open
- Closes when clicking outside or pressing Escape
- Bottom drawer includes drag handle by default
- Smooth slide animations from the specified direction

---

## Accessibility

- Uses `role="dialog"` with proper ARIA attributes
- Focus is trapped within the drawer when open
- Escape key closes the drawer
- Supports keyboard navigation
