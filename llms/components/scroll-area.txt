# Scroll Area Component

Custom scrollable area with styled scrollbars.

---

## Overview

The Scroll Area component provides a scrollable container with custom-styled scrollbars that match your application's design. It supports both vertical and horizontal scrolling with configurable visibility options.

---

## Package

- **Package:** BlazorUI.Components
- **Namespace:** BlazorUI.Components.ScrollArea
- **Source:** https://github.com/blazorui-net/ui/tree/main/src/BlazorUI.Components/Components/ScrollArea

---

## Installation

```bash
dotnet add package BlazorUI.Components
```

Add to `_Imports.razor`:

```razor
@using BlazorUI.Components.ScrollArea
```

---

## API Reference

### Main Component: `<ScrollArea>`

Scrollable container with custom scrollbars.

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Orientation` | `ScrollAreaOrientation` | `Vertical` | Scroll direction |
| `Type` | `ScrollAreaType` | `Auto` | Scrollbar visibility mode |
| `Height` | `string?` | `null` | Fixed height (e.g., "200px") |
| `MaxHeight` | `string?` | `null` | Maximum height |
| `Class` | `string?` | `null` | Additional CSS classes |
| `ChildContent` | `RenderFragment?` | - | Scrollable content |

### Enums

#### `ScrollAreaOrientation`

| Value | Description |
|-------|-------------|
| `Vertical` | Vertical scrolling only |
| `Horizontal` | Horizontal scrolling only |
| `Both` | Both directions |

#### `ScrollAreaType`

| Value | Description |
|-------|-------------|
| `Auto` | Shows scrollbars when content overflows |
| `Always` | Always shows scrollbars |
| `Hover` | Shows scrollbars on hover |
| `Hidden` | Hides scrollbars but allows scrolling |

## Basic Usage

```razor
<ScrollArea Height="200px" Class="w-[350px] rounded-md border p-4">
    <div class="space-y-4">
        <h4 class="text-sm font-medium">Tags</h4>
        @foreach (var tag in tags)
        {
            <div class="text-sm">@tag</div>
            <Separator />
        }
    </div>
</ScrollArea>

@code {
    private string[] tags = { "React", "Vue", "Angular", "Svelte", "Blazor",
        "Next.js", "Nuxt", "Gatsby", "Remix", "Astro" };
}
```

## Examples

### Example 1: Vertical Scrolling

```razor
<ScrollArea Height="200px" Class="rounded-md border">
    <div class="p-4">
        @for (int i = 0; i < 50; i++)
        {
            <p class="text-sm">Item @(i + 1)</p>
        }
    </div>
</ScrollArea>
```

### Example 2: Horizontal Scrolling

```razor
<ScrollArea Orientation="ScrollAreaOrientation.Horizontal"
            Class="w-full whitespace-nowrap rounded-md border">
    <div class="flex w-max space-x-4 p-4">
        @for (int i = 1; i <= 10; i++)
        {
            <div class="shrink-0">
                <div class="h-[150px] w-[250px] bg-muted rounded-md
                            flex items-center justify-center">
                    <span class="text-2xl font-semibold">Image @i</span>
                </div>
            </div>
        }
    </div>
</ScrollArea>
```

### Example 3: With MaxHeight

```razor
<ScrollArea MaxHeight="300px" Class="rounded-md border p-4">
    <div class="space-y-4">
        <h4 class="text-sm font-medium">Lorem Ipsum</h4>
        @for (int i = 0; i < 5; i++)
        {
            <p class="text-sm text-muted-foreground">
                Lorem ipsum dolor sit amet, consectetur adipiscing elit.
                Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
            </p>
        }
    </div>
</ScrollArea>
```

### Example 4: Always Show Scrollbars

```razor
<ScrollArea Type="ScrollAreaType.Always" Height="200px">
    <div class="p-4">
        <!-- Content that may or may not overflow -->
    </div>
</ScrollArea>
```

### Example 5: Both Directions

```razor
<ScrollArea Orientation="ScrollAreaOrientation.Both"
            Height="300px"
            Class="rounded-md border">
    <div class="w-[600px] p-4">
        <!-- Wide and tall content -->
    </div>
</ScrollArea>
```

## Styling

The scrollbars are styled to match your theme:
- Thin, unobtrusive design
- Smooth hover and drag states
- Proper contrast in light and dark modes
- Rounded corners on the track and thumb

---

## Accessibility

- Preserves native scrolling behavior
- Keyboard scrolling works as expected
- Focus management is maintained
- Touch scrolling is supported
